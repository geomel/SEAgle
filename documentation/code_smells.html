Source Code Smells

SEAgle metrics related to source code smells like God Classes and Feature Envy.

Category: Source Code Smells

Mnemonic:
GOD_CLASS

 God Class

Classes that tend to centralize the intelligence of the system. An instance of a god-class performs most of the work, delegating only minor details to a set of trivial classes and using the data from other classes.

Mnemonic:
DATA_CLASS

 Data Class

Data classes are plain classes with a set of properties and their accessors, without implementing any business functionality. They are indications of poor object oriented design, where data are kept separated from logic.

Mnemonic:
FEATURE_ENVY

 Feature Envy

The Feature Envy design disharmony refers to methods that seem more interested in the data of other classes than that of their own class. These methods access directly or via accessor methods a lot of data of other classes. This might be a sign that the method was misplaced and that it should be moved to another class.
